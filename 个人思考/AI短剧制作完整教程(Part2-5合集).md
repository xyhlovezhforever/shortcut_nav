### 1.1 提示词工程（Prompt Engineering）

**基础提示词模板**：
```

要求：
- 类型：【情感/悬疑/古风/都市】
- 时长：30秒-1分钟
- 目标受众：【女性18-35岁/男性20-40岁】
- 核心冲突：【简述冲突】
- 情感基调：【爽感/虐心/搞笑/感动】

请创作一个符合以上要求的短剧剧本，包含：
1. 场景设定
2. 人物介绍（2-3个主要人物）
3. 完整对话
4. 分镜建议（5-8个镜头）

示例：
假设创作霸总短剧，提示词如下：

你是一个专业的短剧编剧，擅长创作都市言情霸总类的30秒-1分钟短剧。

要求：
- 类型：霸道总裁+灰姑娘

- 核心冲突：女主被欺负，霸总霸气护妻
- 情感基调：爽感MAX

请创作第1集剧本。
```

### 1.2 优化剧本的5个技巧

**技巧1：3秒法则**
```
前3秒必须有：
✅ 冲突
✅ 悬念
✅ 反转
✅ 或强烈视觉冲击

❌ 不要：慢慢铺垫
✅ 要：直接进入高潮
```

**技巧2：每15秒一个爽点**
```
30秒短剧 = 2个爽点
45秒短剧 = 3个爽点
60秒短剧 = 4个爽点

爽点 = 让观众"爽"的点
├─ 反转
├─ 打脸
├─ 霸气回怼
└─ 甜蜜互动
```

**技巧3：结尾留钩子**
```
每集结尾必须让观众想看下一集

方法：
├─ 悬念（他到底是谁？）
├─ 反转（真相曝光）
├─ 冲突升级（更大的危机）
└─ 情感转折（表白/分手）
```

**技巧4：对话简短有力**
```
❌ 差对话："我真的很生气，你居然这样对我，
              我们在一起这么久了..."（啰嗦）

✅ 好对话："滚！"（简洁有力）
```

**技巧5：人设鲜明**
```
每个角色要有明显标签：
男主：霸道、护短、深情
女主：坚强、善良、倔强
女配：嫉妒、刻薄、阴险

通过对话和行为强化人设
```

## 第二章：分镜头脚本设计

### 2.1 什么是分镜？

**分镜 = 把剧本拆解成一个个镜头**

示例：
```
剧本：男主霸气护女主

分镜：
镜头1：女配推搡女主（近景，2秒）
镜头2：女主要摔倒（慢动作，1秒）
镜头3：男主一把接住（特写手部，2秒）
镜头4：男主冷眼看向女配（面部特写，2秒）
镜头5：男主说："动她，就是动我"（中景，3秒）
```

---

### 2.2 常用镜头类型

**远景**：
```
用途：展示环境
示例：豪华别墅全景
```

**全景**：
```
用途：展示人物全身
示例：男主西装革履站立
```

**中景**：
```
用途：人物对话场景
示例：腰部以上镜头
```

**近景**：
```
用途：表情特写
示例：面部表情
```

**特写**：
```
用途：强调细节
示例：眼神、手、物品
```

---

### 2.3 Midjourney分镜生成提示词

**基础模板**：
```
[镜头类型], [人物描述], [动作], [场景],
[风格], [光线], [情绪]

示例1（霸总登场）：
medium shot, handsome Asian businessman in black
suit, standing confidently, luxury office background,
cinematic style, dramatic lighting, powerful
atmosphere --ar 9:16

示例2（女主委屈）：
close-up, beautiful young Asian woman, tears in eyes,
sad expression, soft lighting, emotional atmosphere,
高清细腻 --ar 9:16

示例3（冲突场景）：
wide shot, two women arguing, modern cafe interior,
cinematic lighting, tense atmosphere --ar 9:16
```

**保持角色一致性技巧**：
```
方法1：使用--seed参数
└─ 生成第一张角色图后，记录seed值
└─ 后续生成加上相同seed

方法2：使用Midjourney的Vary(Subtle)
└─ 生成一张满意的角色图
└─ 点击V1-V4生成相似图
└─ 用Vary(Subtle)微调

方法3：详细描述+参考图
└─ 第一次生成后保存
└─ 后续用图片+提示词生成
```

# Part 3：AI工具实战应用

## 第一章：Runway Gen-2 视频生成

### 1.1 图生视频（Image to Video）

**操作流程**：
```
步骤1：准备分镜图
└─ 用Midjourney生成的图片

步骤2：登录Runway
└─ 访问：runwayml.com
└─ 选择Gen-2

步骤3：上传图片
└─ 点击"Image to Video"
└─ 上传分镜图

步骤4：设置参数
├─ Motion：1-10（运动幅度）
│  └─ 对话场景：1-3（轻微运动）
│  └─ 动作场景：5-8（明显运动）
├─ Camera Motion：镜头运动
│  └─ Zoom in/out, Pan left/right
└─ Duration：4秒/8秒

步骤5：生成视频
└─ 点击Generate
└─ 等待1-2分钟

步骤6：下载
└─ 满意的话，点击Download
```

**优化技巧**：
```
1. 图片质量越高，视频越好
2. 人物表情要明显
3. 避免复杂背景
4. 同一场景多生成几次，选最好的
5. 4秒效果通常比8秒好
```

---

### 1.2 文生视频（Text to Video）

**提示词模板**：
```
[主体], [动作], [场景], [风格], [镜头运动]

示例：
A handsome Asian businessman walking slowly in
luxury office, cinematic style, camera slowly
zooming in, 4K quality

生成效果：男主在办公室走动的视频
```

**适用场景**：
```
✅ 环境镜头（办公室、街道、房间）
✅ 简单动作（走路、转身、点头）
❌ 复杂对话（效果一般）
❌ 激烈动作（容易变形）

建议：对话用图生视频，环境用文生视频
```

## 第二章：剪映完整剪辑流程

```
步骤1：打开剪映
└─ 点击"开始创作"

步骤2：导入视频片段
└─ 点击"导入"

```

---

### 2.2 视频拼接与过渡

```
步骤1：调整时长
└─ 每个片段2-5秒
└─ 根据节奏调整

步骤2：添加转场
└─ 选中两个片段之间
└─ 点击"转场"
└─ 选择合适的转场效果
├─ 叠化（通用）
├─ 闪黑（戏剧性）
├─ 故障（科技感）
└─ 无缝（自然）

时长：0.3-0.5秒
```

---

### 2.3 添加字幕

```
方法1：自动识别（有对话音频）
└─ 点击"文字" → "识别字幕"
└─ 选择语言
└─ 等待识别完成
└─ 调整错误

方法2：手动添加
└─ 点击"文字" → "新建文本"
└─ 输入台词
└─ 调整位置、大小、字体

推荐字体：
├─ 思源黑体（免费商用）
├─ 阿里巴巴普惠体（免费商用）
└─ 站酷系列字体（免费商用）

字幕样式：
├─ 描边：黑色，粗细3-5
├─ 阴影：适当添加
└─ 动画：逐字出现/淡入淡出
```

---

### 2.4 配音

**方法1：AI配音（推荐）**
```
步骤1：选中文本
└─ 点击"文本朗读"

步骤2：选择音色
├─ 女声：温柔、甜美、知性
└─ 男声：磁性、沉稳、霸气

步骤3：调整参数
├─ 语速：1.0-1.2（稍快更有节奏）
├─ 音量：根据需要调整
└─ 音调：0（默认即可）

推荐音色：
├─ 女主：甜美、温柔
├─ 男主：磁性、霸气
└─ 旁白：知性、新闻播报
```

**方法2：自己配音**
```
步骤1：点击"录音"
步骤2：对照台词录制
步骤3：多录几遍选最好的

优点：更真实
缺点：需要好设备和配音技巧
```

---

### 2.5 配乐与音效

**背景音乐**：
```
步骤1：点击"音频"
└─ 选择"音乐"

步骤2：搜索合适的音乐
关键词：
├─ 情感短剧：抒情、钢琴、温柔
├─ 悬疑短剧：紧张、诡异、悬念
├─ 古风短剧：古筝、笛子、古风
└─ 霸总短剧：大气、震撼、高级感

步骤3：调整音量
└─ 背景音乐：20-40%
└─ 对话时降低：10-20%
```

**音效**：
```
常用音效：
├─ 门开关：嘎吱
├─ 脚步声：咔咔咔
├─ 碰撞：砰
├─ 心跳：咚咚咚
├─ 紧张：咻~
└─ 反转：铛！

添加位置：
└─ 关键动作发生时
└─ 增强代入感
```

---

### 2.6 调色与特效

**调色**：
```
步骤1：点击"调节"

步骤2：选择滤镜
├─ 情感短剧：温暖、柔和
├─ 悬疑短剧：冷色调、低饱和
├─ 霸总短剧：高级灰、商务感
└─ 古风短剧：古典、怀旧

步骤3：手动微调
├─ 亮度：+10 to +20
├─ 对比度：+10 to +15
├─ 饱和度：+5 to +10
└─ 锐化：+10
```

**特效**：
```
常用特效：
├─ 慢动作：重要时刻
├─ 快进：过渡场景
├─ 画面抖动：冲击感
├─ 闪白：时空转换
└─ 模糊：回忆/梦境

使用原则：
⚠️ 不要过度使用
⚠️ 只在关键时刻用
```

# Part 4：制作流程与发布

## 第一章：完整制作SOP

### 标准制作流程（1集）

**Day 1 上午：剧本（2小时）**
```
1. 确定主题和类型
2. 用ChatGPT生成初稿
3. 自己优化剧本
4. 写分镜脚本
```

**Day 1 下午：素材生成（3小时）**
```
1. Midjourney生成人物图（5-10张）
2. Runway生成视频片段（5-8个）
3. 筛选最佳素材
4. 准备音乐和音效
```

**Day 2 上午：剪辑（2小时）**
```
1. 导入素材到剪映
2. 视频拼接
3. 添加字幕
4. AI配音
```

**Day 2 下午：优化与发布（1小时）**
```
1. 配乐和音效
2. 调色和特效
3. 预览检查
4. 导出视频
5. 发布到平台
```

**总耗时：约8小时/集**

**熟练后可以压缩到：4-6小时/集**

## 第三章：数据分析与优化

### 3.1 关键指标

**播放量**：
```
新账号：
├─ 100-500：正常
├─ 500-2000：不错
└─ 2000+：优秀

成熟账号：
├─ 5000-2万：正常
├─ 2万-10万：不错
└─ 10万+：爆款
```

**完播率**（最重要）：
```
30秒视频：
├─ >50%：优秀
├─ 30-50%：良好
└─ <30%：需优化

60秒视频：
├─ >40%：优秀
├─ 20-40%：良好
└─ <20%：需优化

提升方法：
✅ 前3秒抓眼球
✅ 节奏紧凑
✅ 每15秒一个爽点
✅ 结尾留悬念
```

**点赞率**：
```
目标：>5%
计算：点赞数/播放量

提升方法：
└─ 内容质量
└─ 情感共鸣
└─ 引导点赞（口播/字幕）
```

**评论率**：
```
目标：>1%

提升方法：
└─ 设置争议话题
└─ 提问式结尾
└─ 回复评论引导讨论
```

---

### 3.2 根据数据优化

**如果完播率低**：
```
问题分析：
├─ 开头不够吸引？
├─ 节奏拖沓？
├─ 剧情无聊？
```
