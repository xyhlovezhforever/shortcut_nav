## 一、什么是提示词工程?

### 1.1 定义

**提示词工程(Prompt Engineering)** 是一门让AI理解你的需求,并精准输出你想要内容的技术。

**核心思想:**
```
清晰的输入 → 精准的输出
模糊的输入 → 随机的输出
```

### 1.2 为什么需要提示词工程?

**问题场景:**
- ❌ AI输出的内容不是你想要的
- ❌ 需要反复修改才能得到满意结果
- ❌ 每次结果都不一样,不稳定
- ❌ AI理解错误,答非所问

**解决方案:**
- ✅ 掌握提示词工程
- ✅ 用结构化的方式描述需求
- ✅ 给AI明确的角色、任务、要求
- ✅ 一次性得到满意结果

## 二、提示词的黄金法则

### 法则1: 具体 > 模糊

**❌ 错误示例:**
```
帮我写一篇文章
```

**✅ 正确示例:**
```
帮我写一篇1500字的小红书种草文章,
主题是"平价护肤品推荐",
目标读者是18-25岁大学生女性,
风格要口语化、真实接地气,
包含5个产品推荐,每个产品写使用感受和价格。
```

**对比:**
- 模糊需求:AI不知道写什么、怎么写、写多长
- 具体需求:AI清楚所有细节,输出精准

---

### 法则2: 结构化 > 随意描述

**❌ 错误示例:**
```
我想做一个短视频,内容是教别人做菜,你帮我想想怎么拍
```

**✅ 正确示例:**
```
【任务】设计一个60秒美食短视频脚本
【主题】教做番茄炒蛋
【目标平台】抖音
【受众】厨房新手
【要求】
1. 分镜清晰(4-6个镜头)
2. 每个步骤配文字说明
3. 强调简单易学
4. 结尾引导点赞关注

【输出格式】
镜头1: [时长] [画面] [文案]
镜头2: ...
```

**对比:**
- 随意描述:AI不知道从哪开始
- 结构化:AI按框架逐项输出

---

### 法则3: 给示例 > 空想象

**❌ 错误示例:**
```
帮我写10个吸引人的标题
```

**✅ 正确示例:**
```
帮我写10个吸引人的标题,风格参考以下示例:

示例1: "月薪5000,我是如何存下3万的?"
示例2: "这5个习惯,让我效率提升10倍!"
示例3: "别再买了!这些网红产品都是智商税"

要求:
- 使用疑问句或感叹句
- 包含数字
- 有悬念或冲突
- 10-20个字

主题: 职场新人理财
```

**对比:**
- 无示例:AI不知道你的风格偏好
- 有示例:AI模仿示例生成同类内容

---

### 法则4: 分步执行 > 一次完成

**❌ 错误示例:**
```
帮我策划一个完整的短剧,包括剧本、角色、场景、分镜、拍摄方案
```

**✅ 正确示例:**
```
【第一步】先帮我设计3个短剧选题,每个用一句话概括剧情
【第二步】我选定后,再展开详细剧本
【第三步】剧本确定后,再设计角色和场景
【第四步】最后生成分镜脚本

现在先完成第一步。
```

**对比:**
- 一次完成:信息量大,容易出错
- 分步执行:可迭代优化,结果更精准

---

### 法则5: 设定角色 > 直接提问

**❌ 错误示例:**
```
怎么提高写作能力?
```

**✅ 正确示例:**
```
你是一位资深写作导师,有10年教学经验,
擅长帮助新手快速提升写作能力。

我是一个写作新手,想提高文案写作能力,
但不知道从哪里开始。

请你:
1. 分析新手写作的3个核心问题
2. 给出具体的提升方法(每个方法举例说明)
3. 设计一个30天的练习计划

请用简单易懂的语言,避免专业术语。
```

**对比:**
- 直接提问:AI给泛泛而谈的建议
- 设定角色:AI以专家身份给专业建议

## 三、提示词核心框架

### 3.1 CRISPE框架(最常用)

```
C - Capacity (角色设定)
R - Request (具体任务)
I - Input (背景信息)
S - Steps (执行步骤)
P - Parameters (参数要求)
E - Examples (参考示例)
```

#### 实际应用示例

**任务:** 写一篇小红书种草笔记

```
【C - Capacity 角色】
你是一位小红书爆款博主,擅长写真实接地气的种草笔记,
粉丝20万+,笔记平均点赞3000+。

【R - Request 任务】
请帮我写一篇护肤品种草笔记。

【I - Input 背景信息】
产品: 某某品牌玻尿酸精华液
价格: 99元/30ml
我的肤质: 干皮,容易脱皮
使用感受: 吸收快,不黏腻,用了一周皮肤明显水润

【S - Steps 执行步骤】
1. 开头: 吸引眼球的痛点或疑问
2. 中间: 产品介绍+使用感受+效果对比
3. 结尾: 总结+购买建议+引导互动

【P - Parameters 参数】
- 字数: 300-500字
- 风格: 口语化,像朋友聊天
- 必须包含: emoji表情,分段清晰
- 避免: 夸张宣传,假大空

【E - Examples 参考示例】
标题风格参考: "干皮救星!这瓶精华让我告别脱皮"
开头示例: "姐妹们!干皮真的太难了,一到秋冬就脱皮..."

请开始创作。
```

---

### 3.2 RTF框架(适合创意任务)

```
R - Role (角色)
T - Task (任务)
F - Format (格式)
```

#### 示例:短视频脚本

```
【Role 角色】
你是一位短视频导演,擅长设计吸引人的短视频脚本,
作品多次登上抖音热门。

【Task 任务】
为一个美食账号设计一个60秒的短视频脚本,
教观众做"番茄炒蛋"。

【Format 格式】
输出格式:
| 时间 | 镜头 | 画面内容 | 文案/旁白 | 音效 |
|------|------|----------|-----------|------|
| 0-5s | 特写 | ...      | ...       | ...  |

要求:
- 节奏快,每个镜头3-5秒
- 前3秒要抓眼球
- 结尾引导点赞
- 总共10-12个镜头
```

---

### 3.3 SMART框架(适合目标类任务)

```
S - Specific (具体明确)
M - Measurable (可衡量)
A - Achievable (可实现)
R - Relevant (相关性)
T - Time-bound (有时限)
```

#### 示例:学习计划

```
【Specific 具体】
我要学习AI绘画(Midjourney),从零基础到能独立创作。

【Measurable 可衡量】
目标: 30天内掌握Midjourney基础操作,能生成10张高质量作品。

【Achievable 可实现】
我每天可以投入2小时学习和实践。

【Relevant 相关】
我是自媒体博主,需要用AI绘画制作配图和封面。

【Time-bound 时限】
30天内完成。

请帮我设计一个30天的学习计划,包括:
1. 每日学习内容和任务
2. 需要掌握的知识点
3. 实践作业
4. 检验标准

输出格式:
Week 1: [主题]
- Day 1: [学习内容] [实践任务]
- Day 2: ...
```

## 四、进阶技巧

### 4.1 链式思考(Chain of Thought)

**原理:** 让AI展示推理过程,提高复杂任务准确率。

**关键词:** "一步步思考"、"分析过程"、"逐步推理"

**示例:**

```
请一步步分析以下问题:

【问题】
我想做一个抖音账号,但不知道做什么领域。

【我的情况】
- 全职宝妈,带2岁孩子
- 爱做饭、做手工
- 时间不多,每天只有2小时
- 希望能赚点钱补贴家用

【要求】
请一步步帮我分析:
1. 列出所有可能的方向
2. 根据我的情况,筛选出最适合的3个
3. 对比这3个方向的优劣
4. 给出最终建议,并说明理由

请展示你的分析思路。
```

---

### 4.2 角色扮演增强

**技巧:** 给AI设定详细的角色背景,让输出更专业。

**模板:**
```
你是一位 [职业],
拥有 [经验年限] 的从业经验,
擅长 [专长领域],
曾经 [成功案例],
你的风格是 [风格特点]。

现在,请用你的专业知识帮我...
```

**示例:**

```
你是一位资深短视频运营专家,
拥有5年抖音运营经验,
擅长从0到1打造账号,
曾帮助30+账号涨粉到10万+,
你的风格是实战派,不讲虚的,直接给干货。

现在,我的抖音账号发了20条视频,
播放量都在200-500之间,不温不火。

请用你的经验帮我诊断问题,
并给出具体的优化方案。

我的账号是美食类,发的都是家常菜教程。
```

---

### 4.3 Few-shot Learning(少样本学习)

**原理:** 给AI几个示例,让它理解你的需求格式。

**格式:**
```
请按照以下示例格式生成内容:

【示例1】
输入: ...
输出: ...

【示例2】
输入: ...
输出: ...

【示例3】
输入: ...
输出: ...

现在,请用相同格式处理:
输入: [你的内容]
```

**实际应用:**

```
请按照以下示例,为我生成小红书标题:

【示例1】
主题: 减肥
标题: "3个月瘦20斤!我是怎么做到的?(附食谱)"

【示例2】
主题: 护肤
标题: "平价护肤品推荐!学生党闭眼入!(真实测评)"

【示例3】
主题: 穿搭
标题: "小个子女生穿搭秘籍!160也能穿出大长腿"

现在,请用相同格式生成:
主题: 职场新人理财
(生成5个标题)
```

---

### 4.4 负面提示(告诉AI不要做什么)

**技巧:** 明确告诉AI避免什么,防止不想要的输出。

**模板:**
```
【要求】
✅ 做到: ...
❌ 避免: ...
```

**示例:**

```
帮我写一篇职场文章:

【主题】如何提高工作效率

【要求】
✅ 做到:
- 给出具体可操作的方法
- 举真实案例
- 语言简洁,不废话
- 提供工具推荐

❌ 避免:
- 心灵鸡汤,空谈大道理
- 专业术语太多
- 内容太长,超过1500字
- 假大空的建议

【风格】
像朋友聊天,接地气
```

---

### 4.5 迭代优化法

**原理:** 不期望一次完美,通过多次迭代优化。

**流程:**
```
第一次: 生成初稿
第二次: 针对问题优化
第三次: 精修细节
```

**示例:**

```
【第一次提问】
请帮我写一篇小红书种草笔记,
主题是平价精华液推荐,
300字左右。

【AI输出后,第二次提问】
这篇不错,但有3个问题:
1. 开头不够吸引人,改成疑问句
2. 产品介绍太官方,要更口语化
3. 结尾要加上购买渠道

请针对这3点修改。

【AI优化后,第三次提问】
很好!最后请帮我:
1. 加3个emoji表情
2. 把价格加粗显示
3. 增加一个使用前后对比

完成后输出最终版本。
```

## 五、常见场景模板

- 风格: [正式/口语/幽默等]
- 必须包含: [关键信息]
- 避免: [不要的内容]

【参考风格】
[粘贴1-2段参考文字]

【输出格式】
标题: ...
开头: ...
正文: ...
结尾: ...
```

---

### 5.2 创意生成

```
【背景】
我需要为 [项目/产品] 想创意点子。

【目标】
[想达到什么效果]

【限制条件】
- 预算: [金额或无预算]
- 时间: [时限]
- 资源: [有什么资源]

【要求】
1. 生成10个创意点子

   - 核心概念(一句话)

   - 预期效果
   - 难度评估

【风格】
脑洞大开,但要可执行

请开始生成。
```

```
【数据背景】
[粘贴数据或描述数据]

【分析目标】

一、数据概况
二、关键发现
三、问题诊断
四、优化建议
五、行动计划

```

## 六、错误示例与优化

### 案例1: 写小红书标题

#### ❌ 糟糕的提问
```
帮我写10个小红书标题
```

**问题:**
- 没说主题是什么
- 没说风格要求
- 没有参考示例
- AI不知道你的喜好

**AI可能输出:**
```
1. 生活小妙招
2. 今天的穿搭
3. 分享好物
...
(完全没用的标题)
```

---

#### ✅ 优秀的提问

```
请帮我生成10个小红书标题。

【主题】平价护肤品推荐

【目标读者】18-25岁大学生,预算有限

【风格要求】
- 口语化,接地气
- 有悬念或冲突
- 10-20个字
- 包含数字或emoji

【参考示例】
1. "学生党必入!这5款平价精华真的绝了💕"
2. "别再乱买了!这些才是真正的平价好货"
3. "月生活费1000也能养好皮肤?我做到了!"

【要求】
- 每个标题要有吸引力
- 突出"平价"、"学生党"、"性价比"
- 避免浮夸宣传

请开始生成。
```

**AI输出(质量大幅提升):**
```
1. "学生党护肤攻略!100元搞定全套,效果不输大牌💰"
2. "穷学生的平价精华合集,用完皮肤真的变好了!"
3. "这5款平价面霜我回购了N次!便宜又好用"
4. "别被贵妇级坑了!这些平价货才是真香🔥"
5. "月生活费800的我,是怎么养好皮肤的?"
...
```

---

### 案例2: 短视频脚本

#### ❌ 糟糕的提问

```
帮我写一个做菜的视频脚本
```

**问题:**
- 没说做什么菜
- 没说视频时长
- 没说平台和受众
- 没有格式要求

---

#### ✅ 优秀的提问

```
【任务】设计一个美食短视频脚本

【菜品】番茄炒蛋

【平台】抖音

【时长】60秒

【受众】厨房新手

【视频结构】
- 前3秒: 抓眼球(成品特写)
- 3-50秒: 步骤展示(4-5个关键步骤)
- 50-60秒: 成品展示+引导互动

【要求】
1. 每个步骤3-5秒
2. 文字简洁,一句话说明
3. 强调"简单"、"新手也能学会"
4. 配背景音乐节奏点

【输出格式】
| 时间 | 镜头 | 画面内容 | 文案 | 音效/音乐 |
|------|------|----------|------|----------|

## 七、AI工具专用提示词

### 7.1 ChatGPT / Claude

**特点:**
- 适合文本生成、分析、对话
- 可以理解上下文
- 支持多轮对话迭代

**最佳实践:**

```
【第一轮】设定角色和背景

擅长 [专长],

【第二轮】给出具体任务

2. ...

【第三轮】迭代优化
很好,但请优化以下3点:
1. [问题1]
2. [问题2]
3. [问题3]
```

---

### 7.2 Midjourney

**特点:**
- 只能用英文
- 注重视觉描述
- 参数控制很重要

**提示词结构:**
```
[主体描述], [风格], [环境/背景], [光线], [质量参数], [技术参数]
```

**完整模板:**
```
[Main Subject 主体]
A [adjectives] [subject], [age/characteristics],
[clothing/appearance details],
[expression/pose],

[Environment 环境]
[location/setting],
[background details],

[Style & Quality 风格质量]
[art style], [lighting],
[mood/atmosphere],
[quality level], [technical specs],

[Parameters 参数]
--ar [宽高比]
--v [版本]
--s [风格化强度]
--q [质量]
```

**示例:**
```
A beautiful young Chinese woman, 25 years old,
wearing elegant white dress, gentle smile,
long flowing black hair, natural makeup,
standing in a flower garden,
cherry blossoms in background, soft sunlight,
romantic and dreamy atmosphere,
professional photography, cinematic lighting,
ultra realistic, 8k,
--ar 9:16 --v 6 --s 750 --q 2
```

**参数说明:**
- `--ar 9:16`: 竖屏,适合手机
- `--ar 16:9`: 横屏,适合电脑
- `--ar 1:1`: 正方形
- `--v 6`: 版本6(最新)
- `--s 750`: 风格化强度(0-1000)
- `--q 2`: 质量(0.25/0.5/1/2)

### 7.3 Stable Diffusion

**特点:**
- 支持中英文(但英文效果更好)
- 可以精细控制
- 负面提示词很重要

**提示词结构:**
```
【正面提示词】
(quality tags), (subject), (details), (style), (lighting)

【负面提示词】
(things to avoid)
```

**完整示例:**

**正面提示词:**
```
masterpiece, best quality, ultra detailed, 8k,
a beautiful Chinese girl, 20 years old,
long black hair, brown eyes, gentle smile,
wearing white dress, jewelry,
in a garden, flowers, sunlight,
professional photography, soft lighting,
bokeh background, cinematic
```

**负面提示词:**
```
(low quality, worst quality:1.4),
(bad anatomy, bad hands, missing fingers:1.4),
ugly, deformed, blurry, watermark,
text, signature, username,
monochrome, grayscale
```

**关键技巧:**
- 用括号加权重:`(keyword:1.3)` 表示1.3倍权重
- 用中括号减权重:`[keyword:0.8]`
- 逗号分隔关键词
- 越前面的词权重越高

## 八、提示词调试技巧

### 8.1 问题诊断

**如果AI输出不满意,先问自己:**

```
❓ 我的需求够具体吗?
  → 检查是否包含: 主题、风格、长度、格式

❓ 我给了参考示例吗?
  → 没有示例,AI不知道你的偏好

❓ 我设定了角色吗?
  → 有角色,AI输出更专业

❓ 我分步执行了吗?
  → 一次要求太多,容易出错

❓ 我给了负面提示吗?
  → 告诉AI避免什么,减少无效输出
```

---

### 8.2 快速优化法

**步骤1: 复盘问题**
```
AI输出后,先问自己:
- 哪里不满意?
- 具体差在哪?
- 理想状态是什么?
```

**步骤2: 精准反馈**
```
【不要说】
"这个不好,重新写"

【要说】
"这个不错,但有3个问题:
1. [具体问题1] → 改成 [期望1]
2. [具体问题2] → 改成 [期望2]
3. [具体问题3] → 改成 [期望3]

请针对这3点修改,其他保持不变。"
```

**步骤3: 逐步逼近**
```
不期望一次完美,
每次优化1-3个点,
3-5轮后达到理想效果。
```

---

### 8.3 提示词模板库

**建议:**
1. 把好用的提示词保存下来
2. 按场景分类(写作/创意/分析等)
3. 每次使用时,替换具体内容即可
4. 不断迭代优化自己的模板

**模板管理工具:**
- Notion: 建立提示词数据库
- 印象笔记: 按标签分类
- 飞书文档: 团队共享
- 简单的txt文件: 本地保存

## 十、终极检查清单

**发送提示词前,检查这10项:**

```
✅ 1. 角色设定了吗?
   "你是一位..."

✅ 2. 任务具体吗?
   不是"写一篇文章",而是"写一篇1500字的小红书种草文章"

✅ 3. 背景信息完整吗?
   AI需要知道的上下文都给了吗?

✅ 4. 要求明确吗?
   字数、风格、格式都说清楚了吗?

✅ 5. 给示例了吗?
   有参考风格,AI输出更精准

✅ 6. 输出格式指定了吗?
   表格/列表/段落,要说明

✅ 7. 负面提示了吗?
   告诉AI不要什么

✅ 8. 可以分步吗?
   太复杂就分多轮,别一次完成

✅ 9. 有检验标准吗?
   怎样算好?给AI明确标准

✅ 10. 准备好迭代了吗?
   第一次不满意很正常,准备优化
```

## 十一、总结

### 核心要点

1. **具体 > 模糊**
   - 详细描述需求,不要让AI猜

2. **结构化 > 随意**
   - 用框架组织提示词

3. **有示例 > 无示例**
   - 参考风格让AI更懂你

4. **分步 > 一次**
   - 复杂任务拆解执行

5. **迭代 > 完美**
   - 多轮优化,逐步逼近理想结果
