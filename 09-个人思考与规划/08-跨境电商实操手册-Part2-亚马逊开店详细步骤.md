## 第一章：开店前的准备清单

### 1.1 必备材料检查表

在开始注册之前，请确保你已经准备好以下材料：

**✅ 材料清单**：

```
□ 营业执照
  ├─ 类型：个体户或有限公司
  ├─ 状态：有效期内（至少6个月以上）
  ├─ 格式：高清扫描件或照片
  └─ 注意：确保信息清晰可见

□ 法人身份证
  ├─ 正反面照片
  ├─ 有效期内
  ├─ 照片要求：四角完整，信息清晰
  └─ 注意：与营业执照法人一致

□ 双币信用卡
  ├─ 类型：Visa或MasterCard
  ├─ 额度：建议≥5000元
  ├─ 状态：已激活，未过期
  └─ 用途：支付月租费$39.99

□ 收款账户
  ├─ 推荐：连连支付、PingPong、Payoneer
  ├─ 状态：已注册并完成实名认证
  └─ 注意：提前注册，审核需要1-2天

□ 手机号
  ├─ 能接收短信和语音验证码
  ├─ 建议：绑定微信，接收通知更方便
  └─ 注意：一个手机号只能绑定一个账号

□ 邮箱
  ├─ 推荐：Gmail、QQ邮箱、网易邮箱
  ├─ 注意：不要用临时邮箱
  └─ 建议：专门注册一个新邮箱用于亚马逊

□ 电脑
  ├─ 系统：Windows 10/11 或 MacOS
  ├─ 浏览器：Chrome（推荐）
  └─ 网络：稳定的宽带连接
```

### 1.3 避坑指南：账号关联问题

**什么是账号关联？**
- 亚马逊会检测你是否有多个账号
- 如果检测到关联，可能封号
- 新手最容易踩的坑！

**哪些行为会导致关联？**
```
❌ 同一台电脑注册多个账号
❌ 同一个网络IP注册多个账号
❌ 同一个手机号注册多个账号
❌ 同一张信用卡绑定多个账号
❌ 同一个收款账户绑定多个账号
```

**如何避免关联？**
```
✅ 一台电脑只注册一个账号
✅ 一条宽带只注册一个账号
✅ 一个手机号只绑定一个账号
✅ 一张信用卡只绑定一个账号
✅ 一个收款账户只绑定一个账号
```

**如果想做多个店铺怎么办？**
- 方案一：不同的电脑 + 不同的网络 + 不同的资料
- 方案二：使用指纹浏览器（花猫云浏览器、候鸟浏览器）
- 方案三：向亚马逊申请开设多账号（需要理由）

**新手建议**：先做好一个店，不要着急开多店！

## 第三章：KYC视频验证（重要！）

### 3.1 什么是KYC验证？

**KYC = Know Your Customer（了解你的客户）**

**为什么要做视频验证？**
- 亚马逊为了防止虚假注册
- 确认你是真实的卖家
- 不是所有人都需要，随机抽查

**谁会被要求视频验证？**
```
可能性较大的情况：
├─ 新注册账号
├─ 材料审核有疑问
├─ 随机抽查
└─ 敏感地区注册

概率：约30-50%的新卖家会被要求
```

---

### 3.2 收到视频验证邮件后怎么办？

**邮件内容**：
```
主题：Additional information required for your account
内容：We need you to complete a video call verification...
```

**时限**：
- 收到邮件后24小时内完成
- 超时可能导致账号暂停

**第一反应**：
```
✅ 不要慌！这是正常流程
✅ 认真准备，一次通过
❌ 不要忽略邮件
❌ 不要拖延
```

---

### 3.3 视频验证准备清单

```
□ 电脑（带摄像头和麦克风）
```


**环境要求**：
```
✅ 背景整洁
```

### 3.4 视频验证流程（完整版）

**Step 1：预约时间**
```
1. 点击邮件中的链接
2. 选择验证时间
   ├─ 时区：选择 GMT+8（北京时间）
   └─ 时间段：选择你方便的时间
3. 确认预约
4. 收到确认邮件
```

**Step 2：准时进入**
```
1. 提前5分钟打开链接
2. 允许浏览器访问摄像头和麦克风
3. 检查：
   ├─ 摄像头能看到你的脸
   ├─ 麦克风能听到你的声音
   └─ 网络连接稳定
4. 等待客服接入
```

**Step 3：身份确认**
```
客服会要求你：
1. 出示身份证
   └─ 举在摄像头前，确保清晰

2. 出示营业执照
   └─ 举在摄像头前，确保清晰

3. 回答问题（常见问题）：
   Q: What's your name?
   A: My name is Zhang San.

   Q: What's your business name?
   A: My business name is XX Trading Firm.

   Q: What do you plan to sell?
   A: I plan to sell pet supplies / kitchen tools / phone accessories.

   Q: Where is your business located?
   A: My business is located in Beijing, China.
```

**Step 4：手写验证（可能需要）**
```
客服可能要求你：
1. 在纸上手写一串数字
2. 举起来给摄像头看
3. 这是为了确认是真人，不是录像

示例：
客服：Please write down this number: 8372659
你：在纸上写下 8372659，举起来展示
```

**Step 5：确认信息**
```
客服会确认：
├─ 你的姓名
├─ 公司名称
├─ 注册地址
└─ 要销售的产品类目

你只需要回答：Yes, that's correct.
```

**Step 6：结束验证**
```
客服：Thank you, verification is complete.
你：Thank you. Have a nice day!

验证后：
├─ 24-48小时内收到结果
└─ 通过后可以正常使用账号
```

### 3.5 视频验证常见问题与应对

#### Q1：英语不好怎么办？

**答案：没关系！准备好模板即可！**

**实用英语模板**：
```
自我介绍：
Hello, my name is [你的名字拼音].

确认信息：
Yes, that's correct.
No, that's not correct.

听不清时：
Sorry, could you please repeat that?
Pardon? / Sorry?

需要时间思考：
Could you give me a moment?

一切顺利：
Everything is fine.

感谢结束：
Thank you for your help. Have a nice day!
```

**最常见的3个问题**：
```
1. What's your name?
   → My name is [拼音].

2. What will you sell?
   → I will sell [pet supplies / kitchen tools / etc].

3. Can you show me your ID?
   → Sure. Here it is. (举起身份证)
```

---

#### Q2：网络卡顿怎么办？

**预防措施**：
```
✅ 使用有线网络（不要用WiFi）
✅ 关闭其他占网络的程序
✅ 提前测试网速
✅ 准备手机热点作为备用
```

**实际卡顿时**：
```
说：Sorry, the connection is not stable.
   Could we continue in a moment?

客服通常会：
├─ 等待你网络恢复
└─ 或重新预约时间
```

---

#### Q3：没准备好营业执照原件怎么办？

**实在没有的话**：
```
选项1：
├─ 展示电子版（高清照片）
└─ 说明：The original is not with me,
         but I have a clear copy.

选项2：
├─ 请求重新预约

      Could we reschedule?

建议：一定提前准备好原件！
```

```
└─ 材料不全
```

**结果**：
```
1. 收到邮件说明原因
2. 可以重新预约
3. 准备好材料后再次验证
```

**如何避免失败**：
```
✅ 认真准备
✅ 检查所有材料
✅ 测试设备和网络
✅ 提前练习英语回答
```

## 第四章：绑定收款账户

### 4.1 为什么要绑定收款账户？

**作用**：
- 接收亚马逊的销售款项
- 没有绑定就不能提现

**什么时候绑定？**
- 账号审核通过后立即绑定
- 开始销售前必须完成

---

### 4.2 绑定连连支付（详细步骤）

**前提**：连连支付账号已注册并审核通过

**Step 1：登录亚马逊后台**
```
1. 访问：sellercentral.amazon.com
2. 登录你的卖家账号
```

**Step 2：进入收款设置**
```
路径：
Settings（设置）
→ Account Info（账户信息）
→ Deposit Methods（收款方式）
→ Assign（分配）
```

**Step 3：选择收款服务商**
```
1. 点击"Add deposit method"
2. 选择：Lianlian Global（连连支付）
3. 点击"Continue"
```

**Step 4：授权连连支付**
```
1. 会跳转到连连支付页面
2. 登录你的连连账号
3. 同意授权
4. 跳转回亚马逊
```

**Step 5：确认绑定**
```
1. 显示绑定成功
2. 可以看到收款账户信息
3. 设置为主要收款方式
```

**Step 6：测试**
```
亚马逊会发送一小笔测试款项（$0.01-$1）
├─ 1-3个工作日到账
└─ 确认收到后绑定完成
```

---

### 4.3 收款账户常见问题

#### Q1：可以更换收款账户吗？

**答案：可以，但有条件**

```
条件：
├─ 账户必须激活14天以上
├─ 没有正在处理的付款
└─ 新账户通过验证

操作：
└─ 按照绑定流程重新操作即可
```

---

#### Q2：收款账户被封会影响亚马逊账号吗？

**答案：不会直接影响，但会影响收款**

**应对办法**：
```
1. 立即联系收款账户客服
2. 解决被封问题
3. 或更换其他收款账户
4. 确保能正常收款
```

---

#### Q3：多久可以提现一次?

**亚马逊的付款周期**：
```
周期：每14天一次

时间轴：
Day 1: 产生销售
Day 7: 订单完成（客户收货）
Day 14: 钱款到达收款账户
Day 15: 可以提现到银行卡
```

**提现到银行卡**：
```
连连支付：
├─ 提现时间：T+1
├─ 手续费：免费
└─ 到账时间：1个工作日

PingPong：
├─ 提现时间：T+1
├─ 手续费：免费
└─ 到账时间：1个工作日
```

```
│
├─ Payments：付款报表

│
└─ Advertising Reports：广告报表
   └─ 广告效果数据
```

---

#### 功能五：Performance（绩效监控）

**用途**：
- 监控账号健康度
- 查看违规通知
- 提升账号表现

**核心指标**：
```
├─ Order Defect Rate（ODR）：订单缺陷率
│  └─ 要求：< 1%
│
├─ Pre-fulfillment Cancel Rate：取消率
│  └─ 要求：< 2.5%
│

   └─ 要求：< 4%
```

**查看路径**：
```
Performance → Account Health
```

**账号健康度**：
```
绿色：健康
黄色：警告
红色：危险（可能被暂停）
```

## 第六章：常见问题汇总

### Q1：注册过程中信息填错了怎么办？

**答案：看具体情况**

```
可以修改的信息：
├─ 店铺名称
├─ 邮箱（通过客服）
└─ 信用卡

不能修改的信息：
├─ 营业执照
├─ 法人身份证
└─ 企业类型

填错了怎么办？
└─ 只能重新注册（用不同的材料）
```

---

### Q2：审核被拒绝了怎么办？

**常见拒绝原因**：
```
1. 证件不清晰
   └─ 重新上传高清照片

2. 信息不一致
   └─ 检查所有信息是否匹配

3. 材料不全
   └─ 补充缺少的材料

4. 怀疑关联
   └─ 检查是否有重复使用的信息
```

**操作步骤**：
```
1. 查看拒绝邮件，了解原因
2. 准备好正确的材料
3. 重新提交
4. 或联系客服申诉
```

---

### Q3：账号被暂停了怎么办？

**常见原因**：
```
├─ 材料审核未通过
├─ 绩效指标不达标
├─ 涉嫌违规操作
└─ 账号关联
```

**应对方法**：
```
1. 查看邮件了解具体原因
2. 准备申诉材料（POA）
3. 提交申诉
4. 等待审核结果（3-7天）
```

**POA（Plan of Action）模板**：
```
Subject: Appeal for Account Suspension

Dear Amazon Team,

I am writing to appeal the suspension of my account [店铺名称].

1. Root Cause（根本原因）:
   [说明为什么会出现问题]

2. Actions Taken（已采取的行动）:
   [说明你已经做了什么改进]

3. Preventive Measures（预防措施）:
   [说明如何防止再次发生]

I apologize for any inconvenience and promise to comply with
all Amazon policies.

Best regards,
[你的名字]
```

---

### Q4：收到Performance Notification（绩效通知）怎么办？

**不要慌！**

**处理步骤**：
```
1. 仔细阅读邮件
2. 了解具体问题
3. 立即采取行动解决
4. 回复邮件说明改进措施
```

**常见通知类型**：
```
├─ ODR过高：提升产品质量，改善客服
├─ 迟发率高：及时发货，使用FBA
└─ 差评多：优化Listing，改进产品
```
